## 🔄 KPT

### Problem
- 과제의 요구사항을 충족하는 데 집중하여, 해결해야 할 문제 자체에 대해 충분히 고민하지 못했다.
  → 과제의 요구사항을 만족하는 것은 기본이며, 문제를 더 효과적으로 해결할 수 있는 방법이 있는지 고민해야겠다.
- 작성한 코드가 다양한 실행 환경이나 대용량 데이터에서도 안정적으로 동작하는지에 대해서는 고민해보지 않았다.
  → 구현을 마친 후에는 다양한 실행 환경과 데이터 규모를 가정하여 문제가 없는지 점검하는 습관이 필요한 것 같다. 
- 하나의 프로토타입에서 여러 문제를 해결하고자 했다.
  → 하나의 핵심 문제를 해결할 수 있는 것을 우선으로 생각해야겠다.
- EC2를 생성할 때 Amazon Linux를 기본 옵션이라는 이유만으로 선택했다.
  → 작은 선택이라도 해당 기술이 어떤 문제를 해결하기 위해 만들어졌는지와 선택 근거를 먼저 확인한 뒤 결정해야겠다.
- 데이터의 출처와 신뢰성, 품질에 대한 검토가 부족했다.
  → 데이터를 사용하기 전에 데이터의 출처와 수집 방식을 확인하고, 해당 데이터로 어떤 결과를 도출할 수 있는지 충분히 고민해야겠다.
