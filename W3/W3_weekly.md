[잘한 점]
Docker를 활용해 단일 노드부터 멀티노드 Hadoop 클러스터까지 직접 구성하며 분산 환경에 대한 이해를 넓혔다. 
여러 문제가 동시에 발생했을 때는 핵심 데몬의 동작 여부처럼 공통 원인부터 확인하는 방식으로 문제를 해결하려고 노력했다. 
또한 라이브러리 오류가 발생했을 때 예제 코드를 그대로 적용하기보다 내부 동작을 먼저 이해한 뒤 더 단순한 해결 방법을 찾으려 했다. 
워드카운트, 감성 분석, 평점 집계 등 다양한 MapReduce 실습을 반복하며 Hadoop 활용 경험도 쌓을 수 있었다.

[아쉬운 점]
사전 확인 없이 바로 작업을 시작하는 경우가 있었다. 
브랜치 전략이나 PR 조건을 미리 확인하지 않아 작업을 다시 진행한 적이 있었고, 명령이 정상 종료되었다는 이유만으로 서비스가 완전히 준비되었다고 판단해 safe mode나 JobHistory 상태를 확인하지 않고 다음 단계로 넘어가기도 했다. 
또한 root 계정을 기본으로 사용하다 권한 문제를 겪었고, 여러 방법이 가능한 상황에서도 간단한 프로토타입 검증 없이 바로 본 작업을 진행한 점도 아쉬웠다. 
데이터 전처리 없이 분석을 진행해 충분한 인사이트를 얻지 못했고, baseline 없이 결과를 해석해 결론이 다소 모호했던 점도 개선이 필요한 부분이다.
작업을 시작하기 전 필요한 사항을 먼저 확인하고, 실행 후에는 결과와 서비스 상태를 다시 점검하는 습관을 들이고자 한다. 
또한 본 작업에 앞서 프로토타입과 baseline을 먼저 설정하는 절차를 기본으로 실천할 계획이다.
