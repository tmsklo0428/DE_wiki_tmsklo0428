### Problem
- 여러 방법이 있을 때 빠르게 샘플링해서 프로토타입을 먼저 확인하지 않고 바로 실행했음
  → 다음부터는 여러 방법이 있을 때 빠르게 샘플링해서 프로토타입을 먼저 확인한 후 실행하기

- 뭔가를 시켰을 때 그게 지금 어떻게 동작하고 있는지 먼저 확인하지 않아 문제를 놓친 경우가 있었음
  → 다음부터는 뭔가를 시켰을 때 그게 지금 어떻게 동작하고 있는지 먼저 확인하기

- root 계정으로 작업하다가 user 생성 후 권한 문제가 발생함
  → 다음부터는 hdfs_user로 실행하고 필요할 때만 switch해서 root로 테스트하는 방식으로, root 권한을 기본값으로 주지 않기

- jobhistory를 켜지 않은 상태에서 설정값(mapreduce.jobhistory.address)만 확인해 문제 원인을 놓칠 뻔함
  → 다음부터는 설정을 확인하기 전에 관련 서비스가 실제로 켜져 있는지부터 확인하기

- 설정 항목을 xml 파일에 넣어야 하는지 docker-compose.yml에 넣어야 하는지 헷갈려서 시간을 허비함
  → 다음부터는 상황과 버전에 맞게 요청사항의 의도를 먼저 파악하고 설정 위치를 정하기

- 데이터를 전처리 없이 그냥 넣었을 때 가치 있는 인사이트가 나오지 않음
  → 다음부터는 샘플링한 데이터로 먼저 인사이트를 찾아본 후 본작업을 진행하기

- 팀 회고에서 baseline을 정하지 않아 "뭐가 더 좋은 것 같다"는 식의 애매한 결론만 내리게 됨
  → 다음부터는 baseline을 먼저 정하고, 표본/기존 판단이라는 표현을 쓰되 '정답'이라는 단어는 쓰지 않으며, 명확한 결론을 내리고 검증하기
