### Problem
- ResourceManager가 계속 죽는 원인이 signup(세션 종료 시 SIGHUP 전파) 때문임을 늦게 알아차림.
  → 다음부터는 컨테이너 안에서 오래 살아있어야 하는 백그라운드 프로세스는 `-it` 없이, `setsid`/`nohup`으로 세션을 분리해서 실행하는 걸 기본값으로 삼기.
- 명령이 에러 없이 끝났다고 바로 "성공"으로 착각하고 다음 단계로 넘어가서, 데몬이 실제로 살아있는지/서비스가 준비됐는지 재확인하지 않아 문제를 늦게 발견함.
  → 다음부터는 `sleep N && jps`처럼 몇 초 뒤 별도 명령으로 프로세스 생존을 재확인하는 습관 들이기.
- start-dfs.sh 실행 완료 직후 곧바로 검증 스크립트를 돌려서, NameNode가 safe mode(쓰기 차단) 상태인 걸 모르고 HDFS 쓰기에 실패함.
  → 다음부터는 재시작 스크립트에 `hdfs dfsadmin -safemode wait`처럼 서비스 준비 상태를 확인하는 대기 로직을 넣고, "재시작 → 준비 상태 확인 → 다음 단계" 순서로 진행하기.
