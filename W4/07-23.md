## 📝 Review
- **AWS 강의**: EC2·EBS·S3 등 AWS 컴퓨팅/스토리지 서비스의 특징과 차이(가상화 서버, object 기반 저장, region 종속성 등), IAM policy·role의 상속 방식(role은 개인 권한을 덮어씀), 그리고 서버 관리를 최소화하는 서버리스 개념(Lambda의 stateless·cold start·15분 제약, DynamoDB의 AP/CP 일관성)에 대해 배웠다.
- **W4M1**: Docker로 Spark standalone 클러스터(master 1, worker 2)를 구성하고, 샘플 스크립트(pi.py)로 몬테카를로 방식의 π 추정 작업을 spark-submit으로 제출해 Spark Web UI와 로그로 결과를 확인했다.
- **W4M2**: 구성한 Spark 환경에서 NYC TLC 택시 데이터를 Jupyter Notebook으로 불러와 결측치·이상값을 처리하고, 평균 운행 시간·거리, 피크 시간대, 날씨와 수요 간의 관계를 분석했다.
- **W4M2팀 활동**: 이 택시 데이터가 자율주행차 데이터라면 어떤 데이터 프로덕트를 만들 수 있을지 아이디어를 내고 프로토타입을 만드는 팀 활동을 진행했다. 그 결과로 전기차 무인 충전소 입지 추천 프로젝트를 수행해 Vernon Blvd 인근을 최종 후보지로 도출했다.
- **W4M3**: 대규모 웹 크롤러 사례를 통해 병목 지점 분석과 설계 트레이드오프에 대해 학습했다.

## 🔄 KPT

### Problem
- 결측치를 처리할 때 각 항목마다 왜 이런 처리를 하는지 충분히 고민하지 않고 진행하려던 순간이 있었다.
  → 결측치 처리는 항목별로 이유를 먼저 정리한 뒤에 처리 방식을 정해야겠다.
- import문을 코드 맨 위에 두지 않고 작성하다가 나중에 다시 수정했다.
  → 코드를 작성할 때 import문을 무조건 맨 위에 먼저 놓고 시작해야겠다.
- W4M1용 도커를 내리고 W4M2 전용 도커를 새로 띄워 ipynb를 실행했지만, 전체적인 동작 메커니즘이 명확히 이해되지 않았다.
  → 도커 환경을 다시 구성할 때는 단순히 실행에 성공하는 것에서 멈추지 말고, 각 설정이 어떤 역할을 하는지 하나씩 짚어봐야겠다.
